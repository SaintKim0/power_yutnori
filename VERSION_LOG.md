# 모바일 윷놀이 업데이트 로그 (2026-01-07)

오늘 진행된 주요 패치 내역입니다.

## v1.2.0 (시각적 연출 및 메신저 기능 추가)
- **메신저 기능**: 온라인 대전 시 실시간 채팅이 가능한 슬라이드형 메시지창 추가 (자동 숨김 기능 포함).
- **이모지 리액션**: 대전 중 상대방에게 실시간으로 감정을 표현할 수 있는 이모지 애니메이션 추가.
- **연출 강화**: 윷 던지기 역동적 애니메이션 및 말 잡기/완주 시 팡파레(꽃가루) 효과 추가.
- **UX 개선**: 윷 애니메이션 종료 후 결과가 표시되도록 타이밍 조절 및 전반적인 가독성 개선.

## v1.1.13 (혼자하기 모드 버그 수정)
- 온라인 대전 시도 후 '혼자하기'로 돌아왔을 때 팀 명단이 복구되지 않아 게임이 시작되지 않던 치명적 버그 수정.

## v1.1.12 (다시하기 동기화 버그 수정)
- 클라이언트가 먼저 '다시하기'를 요청했을 때 방장 화면에서 투표 결과가 누락되던 이슈 해결.
- 누구나 요청하고 수락하면 즉시 새 게임이 시작되도록 로직 개선.

## v1.1.11 (전역 변수 참조 오류 수정)
- UI 구조 개편 중 누락된 핵심 통신 변수(peer, isHost 등) 복구.
- '방 만들기' 버튼이 반응하지 않던 현상 해결.

## v1.1.10 (온라인 UI 및 안정성 강화)
- 온라인 모드 진입 시 기존 오프라인 명단이 남아있던 문제 해결.
- 방 생성 시 UI가 깜빡이며 복사 버튼이 사라지던 현상 수정.

## v1.1.9 (방 코드 복사 버튼 복구)
- 방 생성 후 6자리 코드와 함께 '링크 복사', '코드 복사' 버튼이 정상적으로 표시되도록 수정.
- 온라인 접속 인원 명단을 실시간으로 확인할 수 있도록 UI 통합.

## v1.1.7 ~ v1.1.8 (프로필 시스템 및 방 생성 로직)
- 로컬 기기 저장 기반 프로필(별명, 아바타 이미지) 기능 추가.
- 온라인 대전 시 설정된 프로필 자동 사용.
- 서버 연결 초기화 과정에서 버튼 클릭이 무시되던 대기 로직 개선.

## v1.1.6 (UI 구조 대개편)
- 메인 메뉴를 '👤 혼자하기'와 '👥 같이하기'로 이원화.
- 같이하기 하위에 '오프라인(다수팀)'과 '온라인 대전' 서브 메뉴 구성.

## v1.1.4 ~ v1.1.5 (상호작용 제한 및 다인용 다시하기)
- 상대방 차례일 때 버튼 및 말 클릭 비활성화 (실수 방지).
- 다인 플레이 시 일부 인원이 나가더라도 남은 인원끼리 게임을 지속할 수 있도록 개선.

## v1.1.1 ~ v1.1.3 (온라인 동기화 및 다시하기 기초)
- 낙장(실패) 상황 실시간 동기화 오류 수정.
- 참가자(Client)가 방장의 상태를 100% 추종하도록 `applySyncState` 구현.
- 게임 종료 후 '다시 하기' 요청 및 수락 시스템 최초 도입.
